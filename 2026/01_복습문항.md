### **I. 참/거짓 형 문항 (10문항)**

1. **문항:** C++에서 함수 오버로딩을 수행할 때, 함수의 반환 타입(Return Type)만 다르고 매개변수의 타입과 개수가 같다면 올바르게 오버로딩이 성립된다.
* **정답:** 거짓
* **해설:** 함수 오버로딩은 매개변수의 타입, 개수, 순서에 의해서만 구별되며 반환 타입만 다른 경우에는 오버로딩을 할 수 없습니다.


2. **문항:** 상속 관계에서 부모 클래스의 포인터로 자식 클래스 객체를 가리키고 있을 때, 가상 함수(`virtual`)가 아닌 일반 함수를 호출하면 부모 클래스에 정의된 함수가 호출된다.
* **정답:** 참
* **해설:** 가상 함수가 아닐 경우 컴파일 시점에 포인터의 타입에 따라 호출될 함수가 결정(정적 바인딩)되므로 부모 클래스의 함수가 실행됩니다.


3. **문항:** 사용자 정의 클래스에서 연산자 오버로딩을 구현할 때, C++에 기존에 존재하지 않던 새로운 기호(예: ``)를 연산자로 정의하여 사용할 수 있다.
* **정답:** 거짓
* **해설:** C++ 연산자 오버로딩은 기존에 존재하는 연산자(`+`, `-`, `*`, `==` 등)에 대해서만 재정의가 가능하며, 새로운 연산자 기호를 만들어낼 수는 없습니다.


4. **문항:** `std::vector`는 동적 배열 컨테이너이므로 요소가 추가될 때마다 매번 메모리 할당과 기존 요소의 복사가 무조건 일어난다.
* **정답:** 거짓
* **해설:** `std::vector`는 내부적으로 여유 공간(`capacity`)을 미리 할당해 두기 때문에, 여유 공간이 채워지기 전까지는 매번 메모리 할당이 일어나지 않습니다.


5. **문항:** 클래스의 멤버 함수 뒤에 `const` 키워드를 붙이면(예: `void show() const`), 해당 함수 내부에서는 클래스의 멤버 변수 값을 수정할 수 없다.
* **정답:** 참
* **해설:** 멤버 함수 뒤의 `const`는 해당 함수가 읽기 전용(Read-only) 함수임을 보장하며 멤버 변수의 수정을 금지합니다.


6. **문항:** `try-catch` 블록을 사용할 때, 부모 클래스 타입의 예외 catch 블록을 자식 클래스 타입의 예외 catch 블록보다 위에 먼저 배치해야 자식 예외를 안전하게 가려낼 수 있다.
* **정답:** 거짓
* **해설:** 부모 클래스 타입의 catch 블록이 위에 있으면 자식 예외 객체도 부모 타입으로 먼저 포착되므로, 반드시 서브클래스(자식) catch 블록을 위에 먼저 배치해야 합니다.


7. **문항:** 스마트 포인터인 `std::unique_ptr`은 독점적 소유권을 가지므로 다른 `std::unique_ptr` 객체로 복사(Copy)할 수 없다.
* **정답:** 참
* **해설:** `unique_ptr`은 유일한 소유권을 보장하기 위해 복사 생성자와 복사 대입 연산자가 제한되어 있으며, 소유권 이전은 이동(`std::move`)을 통해서만 가능합니다.


8. **문항:** C++17에서 도입된 강타입 열거형(`enum class`)은 전통적인 `enum`과 달리 서로 다른 열거형 타입 간의 암시적 정수 변환을 허용하지 않아 타입 안전성이 높다.
* **정답:** 참
* **해설:** `enum class`는 범위(Scope)를 제한하고 암시적 형변환을 막아주기 때문에 안전한 상태 관리가 가능합니다.


9. **문항:** 이동 세만틱(Move Semantics)은 객체가 가진 실제 메모리 자원을 깊은 복사(Deep Copy)하지 않고, 자원의 소유권만 포인터 복사 형태로 빠르게 이전하는 기술이다.
* **정답:** 참
* **해설:** 불필요한 임시 객체의 대규모 메모리 할당 및 복사 오버헤드를 줄여 프로그램의 성능을 크게 최적화합니다.


10. **문항:** SQLite는 파일 기반의 서버리스 데이터베이스이므로, C++ 프로젝트에서 활용하기 위해서는 반드시 PC에 별도의 DB 서버 프로세스를 설치하고 구동해야 한다.
* **정답:** 거짓
* **해설:** SQLite는 별도의 서버 프로세스 없이 소스 파일(`.c`, `.h`)과 로컬 파일 하나만으로 완벽하게 동작하는 독립적 데이터베이스 시스템입니다.

### **II. 괄호 메우기 문항 (10문항)**

11. **문항:** 함수 이름은 같지만 매개변수의 타입이나 개수가 다르게 여러 개 정의하는 기법을 함수 오버로딩이라고 하며, 이와 달리 상속 관계에서 부모의 가상 함수를 자식 클래스에서 재정의하는 것을 함수 ( **오버라이딩** )(이)라고 한다.
12. **문항:** 클래스 내부에 선언되어 객체마다 각각 생성되지 않고 클래스당 단 하나만 존재하며 모든 객체가 공간을 공유하는 변수를 ( **정적 멤버 변수** )(이)라고 한다. (영문 키워드는 static)
13. **문항:** C++에서 자원의 획득은 객체의 초기화 단계에서 이루어져야 하며, 객체가 범위를 벗어나 소멸될 때 자원도 자동으로 해제되어야 한다는 현대적 자원 관리 디자인 패턴을 ( **RAII** )(이)라고 한다.
14. **문항:** 템플릿 프로그래밍이나 현대 C++에서 컴파일러가 초기화 식을 바탕으로 변수의 타입을 자동으로 추론하도록 지정할 때 사용하는 키워드는 ( **auto** )이다.
15. **문항:** C++ 표준 라이브러리(STL)에서 키(Key)와 값(Value)의 쌍으로 데이터를 관리하며, 내부적으로 이진 탐색 트리 구조를 취해 키 값을 기준으로 자동 정렬되어 저장되는 컨테이너는 ( **map** )이다.
16. **문항:** C++ 클래스 내부에서 프라이빗(`private`) 멤버에 외부의 특정 함수나 클래스가 직접 접근할 수 있도록 권한을 부여할 때 사용하는 선언 키워드는 ( **friend** )이다.
17. **문항:** C++17에서 도입된 문법으로, `std::map`의 내부 요소나 구조체 등을 `auto const& [key, val] : my_map`과 같이 개별 변수로 즉시 분해하여 바인딩하는 기능을 ( **구조적 바인딩** )(이)라고 한다.
18. **문항:** 함수 내부나 코드 블록 안에서 이름 없이 즉석으로 정의하여 사용할 수 있는 현대 C++의 익명 함수 표현식을 ( **람다 표현식** )(이)라고 한다.
19. **문항:** Win32 API 그래픽스 및 OpenGL 프로그래밍에서, 화면의 출력 버퍼에 직접 그림을 그릴 때 발생하는 깜빡임 현상을 방지하기 위해 보이지 않는 백 버퍼(Back Buffer)에 먼저 렌더링한 후 화면을 교체하는 기술을 ( **더블 버퍼링** )(이)라고 한다.
20. **문항:** 컴파일 타임에 해당 함수나 변수의 연산이 상수로 평가되도록 강제하여 프로그램 실행 시점의 연산 속도를 극대화하는 최적화 키워드는 ( **constexpr** )이다.
