# 문제

## I. 참/거짓 형 문항

1. C++에서 함수 오버로딩을 수행할 때, 함수의 반환 타입(Return Type)만 다르고 매개변수의 타입과 개수가 같다면 올바르게 오버로딩이 성립된다.

2. 상속 관계에서 부모 클래스의 포인터로 자식 클래스 객체를 가리키고 있을 때, 가상 함수(`virtual`)가 아닌 일반 함수를 호출하면 부모 클래스에 정의된 함수가 호출된다.

3. 사용자 정의 클래스에서 연산자 오버로딩을 구현할 때, C++에 기존에 존재하지 않던 새로운 기호를 연산자로 정의하여 사용할 수 있다.

4. `std::vector`는 동적 배열 컨테이너이므로 요소가 추가될 때마다 매번 메모리 할당과 기존 요소의 복사가 무조건 일어난다.

5. 클래스의 멤버 함수 뒤에 `const` 키워드를 붙이면(예: `void show() const`), 해당 함수 내부에서는 클래스의 멤버 변수 값을 수정할 수 없다.

6. `try-catch` 블록을 사용할 때, 부모 클래스 타입의 예외 catch 블록을 자식 클래스 타입의 예외 catch 블록보다 위에 먼저 배치해야 자식 예외를 안전하게 가려낼 수 있다.

7. 스마트 포인터인 `std::unique_ptr`은 독점적 소유권을 가지므로 다른 `std::unique_ptr` 객체로 복사(Copy)할 수 없다.

8. C++17에서 도입된 강타입 열거형(`enum class`)은 전통적인 `enum`과 달리 서로 다른 열거형 타입 간의 암시적 정수 변환을 허용하지 않아 타입 안전성이 높다.

9. 이동 세만틱(Move Semantics)은 객체가 가진 실제 메모리 자원을 깊은 복사(Deep Copy)하지 않고, 자원의 소유권만 포인터 복사 형태로 빠르게 이전하는 기술이다.

10. SQLite는 파일 기반의 서버리스 데이터베이스이므로, C++ 프로젝트에서 활용하기 위해서는 반드시 PC에 별도의 DB 서버 프로세스를 설치하고 구동해야 한다.

13. C++에서 자원의 획득은 객체의 초기화 단계에서 이루어져야 하며, 객체가 범위를 벗어나 소멸될 때 자원도 자동으로 해제되어야 한다는 현대적 자원 관리 디자인 패턴을 (          )이라고 한다.

15. C++ 표준 라이브러리(STL)에서 키(Key)와 값(Value)의 쌍으로 데이터를 관리하며, 내부적으로 이진 탐색 트리 구조를 취해 키 값을 기준으로 자동 정렬되어 저장되는 컨테이너는 (          )이다.

16. C++ 클래스 내부에서 프라이빗(`private`) 멤버에 외부의 특정 함수나 클래스가 직접 접근할 수 있도록 권한을 부여할 때 사용하는 선언 키워드는 (          )이다.

17. C++17에서 도입된 문법으로, `std::map`의 내부 요소나 구조체 등을 `auto const& [key, val] : my_map`과 같이 개별 변수로 즉시 분해하여 바인딩하는 기능을 (          )이라고 한다.

18. 함수 내부나 코드 블록 안에서 이름 없이 즉석으로 정의하여 사용할 수 있는 현대 C++의 익명 함수 표현식을 (          )이라고 한다.

19. Win32 API 그래픽스 및 OpenGL 프로그래밍에서, 화면의 출력 버퍼에 직접 그림을 그릴 때 발생하는 깜빡임 현상을 방지하기 위해 보이지 않는 백 버퍼(Back Buffer)에 먼저 렌더링한 후 화면을 교체하는 기술을 (          )이라고 한다.

20. 컴파일 타임에 해당 함수나 변수의 연산이 상수로 평가되도록 강제하여 프로그램 실행 시점의 연산 속도를 극대화하는 최적화 키워드는 (          )이다.
